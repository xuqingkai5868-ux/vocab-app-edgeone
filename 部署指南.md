# 部署到 EdgeOne Pages 完整指南

> 适用：暑假共学背单词系统（3 账号 PIN 鉴权 + 1460 词 + 83 天计划）
> 部署目标：拿到 `https://scdq.online` 这种国内可访问的永久域名
> 预计耗时：30-45 分钟（你 + 我一起做）
> 上一次部署：2026-06-04，从 Vercel 迁来（解决国内无法访问的问题）

---

## 架构一句话

```
浏览器 (index.html)
  ↓ https://scdq.online
EdgeOne Pages（静态托管 + Edge Functions）
  ↓
EdgeOne KV 命名空间（my_kv 全局变量，3 个 key 族：user / pin / state / session）
```

---

## 你需要做的事（5 步，每步 3-8 分钟）

### Step 1：EdgeOne 控制台创建 Pages 项目

1. 打开 https://console.cloud.tencent.com/edgeone/pages
2. 点「创建项目」→ 项目名填 `vocab-app`（这决定你的默认域名前缀）
3. **关键**：加速区域选 **「中国大陆可用区」**（不是「全球可用区」）
   - 中国大陆可用区 = 国内访问快，但**必须 ICP 备案过的域名**才能绑
   - 我们已经备案过了（scdq.online），所以选这个
4. 其他默认 → 创建

> ⚠️ 如果你之前在 EdgeOne 已经有 `eustudy` 项目并选了「全球可用区」，那个项目国内访问会 401。
> **建议直接在控制台删掉 eustudy 项目，从头开一个 vocab-app**（也 5 分钟）。

### Step 2：创建并绑定 KV 命名空间

1. 同一控制台 → 左侧「存储」→ 「KV 存储」→ 「创建命名空间」
2. 名字填 `vocab-kv`（随便，代码里不用这个名字）
3. 创建后 → 找到这个命名空间 → 右侧「绑定到 Pages」
4. 绑到刚才创建的 `vocab-app` 项目
5. **关键**：绑定时弹窗会问"变量名"，**填 `my_kv`**（一字不差）
   - 这个名字会作为全局变量注入到 Edge Function
   - 代码里 `await my_kv.get(key)` 就是这个变量
6. 绑定完成

> 如果绑定时填错了变量名，Edge Function 里 `my_kv` 会是 undefined，调用时会报 `my_kv 未绑定`。

### Step 3：本地部署（CLI 推代码）

打开 PowerShell（**在 `C:\CODE\vocab-app\` 目录下**）：

```powershell
# 第一次部署：会跳浏览器授权
npx -y edgeone@latest pages login

# 部署到生产环境
npx -y edgeone@latest pages deploy
```

部署成功后控制台会显示：
- ✅ 默认域名：`vocab-app.edgeone.app`（**这个国内 401，无法用**）
- ✅ 你绑定的 scdq.online 域名（接下来 Step 4 配）

### Step 4：绑定自定义域名 scdq.online

1. EdgeOne 控制台 → 你的 `vocab-app` 项目 → 「自定义域名」 tab
2. 点「添加域名」→ 填 `scdq.online`（也建议加 `www.scdq.online`）
3. EdgeOne 会给你：
   - **CNAME 记录**（形如 `vocab-app.edgeone.app.cdn.cloudflare.net`）——这个是要配的
   - **TXT 记录**（域名验证用）——这个也要配
4. 登录 DNSPod（https://console.dnspod.cn）→ 找到 `scdq.online` 域名 → 「记录管理」：
   - 添加 `scdq.online` 的 CNAME → 指向 EdgeOne 给你的 CNAME 值 → 代理关掉（DNS only）
   - 添加 `_edgeone-verify.scdq.online` 的 TXT → 内容 = EdgeOne 给的验证串
5. 等 5-10 分钟让 DNS 生效
6. EdgeOne 控制台 → 域名状态会自动从「验证中」→「已激活」
7. **HTTPS 证书自动签发**（约 5-10 分钟），不需要你手动操作

### Step 5：初始化 3 账号（一次性的）

EdgeOne KV 是空的，部署完没有账号数据。PowerShell 里跑：

```powershell
curl -X POST https://scdq.online/api/seed `
  -H "Content-Type: application/json" `
  -d '{
    "users": {
      "gao":   { "name": "哥哥", "role": "user", "grade": "高一" },
      "di":    { "name": "弟弟", "role": "user", "grade": "三年级" },
      "admin": { "name": "管理员", "role": "admin" }
    },
    "pins": { "gao": "1234", "di": "5678", "admin": "9999" }
  }'
```

返回 `{"ok":true,"seeded":["gao","di","admin"],"overwritten":false}` = 成功。

### Step 6：测试 3 账号

浏览器打开 `https://scdq.online`：
- 选「哥哥」→ PIN `1234` → 应该进首页，看到 day 1
- 右上角切「弟弟」→ PIN `5678`
- 切「管理员」→ PIN `9999` → 应该能看到所有人的进度
- 标几个词为「掌握」→ 关浏览器 → 再打开同一个账号 → 进度应保留（证明云端 KV 存住了）
- **国内网络直连**（不需要 VPN）→ 这是迁移到 EdgeOne 的核心收益

## 常见问题

### Q：为什么默认域名 `vocab-app.edgeone.app` 国内 401？
A：EdgeOne 默认域名是"预览链接"性质，3 小时过期。要稳定访问**必须绑自定义域名 + ICP 备案**。
我们 `scdq.online` 已经备案过了，所以绑上就能用。

### Q：旧的 Vercel 数据（哥哥弟弟之前的进度）能迁移过来吗？
A：**不能自动迁移**。Vercel KV 和 EdgeOne KV 是两套独立的存储。
如果你要保留旧进度：
1. 浏览器登录 Vercel 版的 `scdq.vercel.app`
2. 进 admin（PIN 9999）→ 进每个用户 → 复制 state 到本地
3. 在 EdgeOne 版 seed 完后，手动 PUT state 回去（需要写个迁移脚本）

**不推荐这样做**，理由：
- 暑假刚开始，1447 词才背到 ~30 词，重来不亏
- 孩子的状态标记（掌握/模糊）本来就不准，重新标更准

### Q：以后改前端要重新部署？
A：是的。`npx -y edgeone@latest pages deploy` 一次。30 秒生效。

### Q：以后加新词要重新部署吗？
A：要。前端 `index.html` 嵌入了 SCHEDULE 数组（83 天词表），改词表就要重新构建+部署。

### Q：怎么换 PIN？
A：浏览器用 admin PIN `9999` 登录，**当前没有 UI 改 PIN**。改用 seed 重置：
```powershell
curl -X POST https://scdq.online/api/seed -H "Content-Type: application/json" -d '{
  "users": {...},
  "pins": {"gao":"新PIN","di":"新PIN","admin":"新PIN"},
  "adminConfirmPin": "9999"
}'
```

### Q：怎么删一个账号？
A：EdgeOne 控制台 → 存储 → KV 命名空间 → 找到 `user:xxx` / `pin:xxx` / `state:xxx` 三个 key → Delete。

### Q：本地开发怎么跑？
A：
```powershell
cd C:\CODE\vocab-app
node local-server.js
# 浏览器开 http://localhost:3000
```
注意 `local-server.js` 用的是内存版 KV（重启清空），**只能测前端，不能测 API**。
要本地测 API 需要装 EdgeOne CLI 的本地模拟器（暂未配置，不在当前范围）。

### Q：部署到一半出错了？
A：把错误信息发给我，常见三类：
1. `my_kv 未绑定` → 控制台没绑 KV 或变量名填错
2. `401 unauthorized` 调用 API → token 没带上（前端会自动重登）
3. `404 Not Found` → edge-functions/api/ 路径不对，重新 deploy 一次

## 当前项目结构

```
vocab-app/
├── middleware.js                          ← EdgeOne Pages 路由 + CORS
├── edge-functions/
│   └── api/
│       ├── _lib/
│       │   ├── kv.js                      ← my_kv 全局变量封装
│       │   ├── respond.js                 ← JSON 响应 + CORS 头
│       │   └── verifyToken.js             ← Bearer Token 验证（30天有效）
│       ├── login.js                       ← POST  验证 PIN，返回 token
│       ├── me.js                          ← GET   返回当前用户资料
│       ├── state.js                       ← GET/PUT  读写学习进度
│       ├── summary.js                     ← GET   所有人进度概览
│       ├── reset.js                       ← POST  重置进度（admin 权限）
│       └── seed.js                        ← POST  初始化 3 账号
├── index.html                             ← 前端单页（直接放 EdgeOne 静态托管）
├── local-server.js                        ← 本地开发服务器（内存版 KV）
├── 样章/                                  ← 100 词样章
├── vocab-批次{1-5}.{csv,md}               ← 1447 词词表
├── plan-完整计划.{csv,md}                 ← 83 天计划
├── schedule.{min,}.json                   ← 词表 JSON 数据
├── 部署指南.md                            ← 本文件
├── 部署指南-Vercel.md                     ← 旧 Vercel 版（保留备查）
├── generate_plan.py                       ← 一次性生成脚本
└── build_html.py                          ← 一次性构建脚本
```

## 关键设计决策记录

| 决策 | 理由 |
|---|---|
| KV 变量名固定 `my_kv` | 控制台绑定时强提示，避免 typo |
| Middleware 拿不到 my_kv | V8 Isolate 设计，所以最终验证放 API 内 |
| seed 端点不需要 token | 首次初始化必须能调；已有管理员后必须用 `adminConfirmPin` 二次保护 |
| 角色二次校验 (3 层) | 后端最强 (verifyToken) + 客户端函数 (doReset) + UI 隐藏 (admin-only) |
| 状态值限定 `mastered` / `fuzzy` | 防止任意字符串污染，方便后续分析 |
| session 30 天过期 + token 落 KV | 换设备登录方便，注销/重置自动失效 |
